Краткая информация о программе

Система контроля доступом – совокупность программно-аппаратных технических средств безопасности, служащая для автоматизации входных групп (шлагбаумы,ворота,калитки и т.д.)жилых дворов, коттеджных поселков, частных домов, предприятий, платных парковок и т.д.

Доступ к шлагбаумам обеспечивается без вмешательства оператора, но оператор имеет доступ к программному обеспечению, содержащему всю информацию об объектах. Внутренний пользователь после подтверждения доступа к системе имеет возможность контролировать въезд для своего транспортного средства через мобильное приложение, SMS, либо с помощью оператора, а также запрашивать допуск для въезда незарегистрированных в системе транспортных средств.

Как работает удаленное управление и диспетчеризация шлагбаума:

  • при вызове оператора, он проверяет наличие пропуска и принимает решение о допуске;
  • в случае проезда транспорта экстренных служб, диспетчер незамедлительно открывает шлагбаум
  • закрытие шлагбаума происходит автоматически

    • Диспетчерская служба выполняет следующие задачи:

  • круглосуточное наблюдение за объектом
  • пропуск гостей (при наличие заказанного пропуска)
  • формирование полноценной статистики по проездам и пользованию устройствами

    • Пользователь (житель):

  • имеет постоянный уникальный код
  • въезд осуществляется путем считывания гос.номера автомобиля, зарегистрированного в системе, либо при наборе кода на кодовой клавиатуре (и/или при наличии RFID- карты)
  • имеет право заказывать пропуска

    • Гость:

  • имеет разовый допуск по коду

    • Мониторинг

    Главная страница сайта должна схематически отображать все автомобили, находящиеся на территории (и,если имеется на подземном паркинге) с указанием времени парковки, общую информацию об автомобилях, нарушевших правила нахождения на территории, текущие пропуска (включая заказанные на текущий день), графическое изображение всех видов проезда на территорию, а так же уведомления о вызове (входящем звонке) с улицы и чат. Жителям модуль не отображается.

    Мониторинг

    Главная страница программного обеспечения

    Общая структура модуля “Мониторинг” содержит:

  • Колонка “Нарушители” ( автомобили, нарушившие правила)
  • Поле ввода уникального кода с кнопкой “Продлить”
  • Поле ввода уникального кода для проверки нарушителей с кнопкой “Найти”
  • Поле ввода уникального кода для разового пропуска с кнопкой “Отправить”
  • Поля с перечнем заказанных/существующих пропусков
  • Статистическая диаграмма посещений
  • Чат

    • Колонка “На территории” отражает список автомобилей, находящихся на территории.

    На зеленой шапке в колонке в белом круге выводится общее количество автомобилей.

    Далее находится поисковая строка.

    Поиск должен работать по всем полям.

    Первая увеличенная строка в колонке показывает автомобиль, который сейчас заезжает на территорию.

    В данной строке 4 поля : ФИО,тип пропуска,телефон,время въезда,а также фотография автомобиля в момент въезда. Под полями находится иконка часы и таймер обратного отсчета времени. и проезда данного автомобиля строка уменьшается. Остается только три поля и таймер. При единичном клике по строке окно увеличивается до первоначального размера.

    Колонка “Пропуска” отражает список пропусков, действительных на текущих день.Поиск должен работать по всем полям.

    Далее в каждой строке находятся текстовые поля : ФИО, квартира, номер машины, тип пропуска и период действия.
  • Поле ввода уникального кода для проверки нарушителей с кнопкой “Найти”
  • Поле ввода уникального кода для разового пропуска с кнопкой “Отправить”
  • Поле ввода уникального кода с кнопкой “Продлить”
  • Чат для личного общения с жителями (зарегистрированными пользователями), а также для рассылки общей информации
  • Статистическая диаграмма посещений

    • Контроль посещений

    Раздел состоит из персональных карточек каждого жителя (сотрудника) с указанием имени, времени первого входа и последнего события, а также фотографией последнего входа.

    Контроль посещений

    Структура модуля

    Над карточками находятся фильтрация и поле поиск.

    В фильтре указывается : “на месте”, “вышел” и “отсутсвует”.

    Карточки “на месте” выделяются зеленым цветом, карточки “вышел”- красным, карточки “отсутствует” - оранжевым.

    Квартиры

    Структура модуля

    Структура левого поля “Список квартир”:

  • зеленая шапка с названием колонки

  • иконка “история” - при нажатии открывается окно история по каждому пользователю

    • При нажатии на поле “Квартира №” в левой колонке в правом поле появляется соответствующая информация.

    Правое поле “Информация о пользователе” включает в себя:

  • зеленая шапка с названием колонки (номер выбранной квартиры)
  • вкладки настройки, жители и аккаунты

    • Выбранная вкладка подчеркивается зеленой полосой, информация в каждой вкладке редактируется отдельно ниже в текстовых полях для ввода и кнопкой “Сохранить”

    Авторизация

    Поле “Электронная почта” работает с валидацией (проверкой) на пустоту: заполненность от 5-ти символов. Обязательное содержание символа @ и . Все данные для проверки тянутся с сервера при первой загрузке.

    При неверном вводе данных приложение выводит notification, в каком из полей возникла ошибка. Например, “Неверно введена почта”. В пустом состоянии поле имеет place holder “Электронная почта”, автоматического фокуса на поле нет. При выставлении фокуса, place holder не скрывается, только начав ввод - пропадает, заменяется на вводимые данные.

    Поле “пароль” показывает значение символа введенного последним, каждый предыдущий скрывается точкой. При нажатии на кнопку “Войти” происходит отправка данных на сервер и получение ответа. Если пользователь входит по автоматически сгенерированному паролю, мы показываем дополнительный экран смены пароля (описано ниже).

    При нажатии на “Забыли пароль” пользователя переводит на новый экран “Восстановления пароля”.

    Восстановление пароля

    Поле “Электронная почта” работает с валидацией на пустоту, длину заполненности от 5-ти символов. Обязательное содержание символа @ и .

    При неверном вводе данных приложение сообщает, в каком из полей возникла ошибка. Например, “Неверно введена почта”. Кнопка “Восстановить” при нажатии на которую передается информация на сервер.

    После успешной отправки информации на сервер, показываем notification с текстом “Новый пароль отправлен на Вашу почту” и кнопка “Ок”, по нажатию на которую сообщение скрывается. Автоматический переход на экран авторизации, в поле e-mail находится почта, которую вводили ранее на экране восстановления пароля.

    Регистрация

    Окно регистрации включает в себя такие поля как:

  • ФИО
  • Электронная почта
  • Пароль
  • Номер телефона

    • Поле “Электронная почта” работает с валидацией на пустоту, длину заполненности от 5-ти символов. Обязательное содержание символа @ и . При неверном вводе данных приложение сообщает в каком из полей возникла ошибка. Например, “Неверно введена почта”.

    По клику на кнопку “Создать пароль”, пользователь на почту получает автоматически сгенерированный пароль. Программа показывает notification “Пароль отправлен на Вашу почту”, по клику на “Ок” автоматически происходит переход на экран авторизации, и ранее введенная почта (для регистрации) автоматически вставлена в поле e-mail на экране авторизации. При вводе автоматически сгенерированного пароля, и авторизации, пользователь попадает на дополнительный экран смены пароля.

    Смена временного пароля

    Проверка работает на совпадение паролей. Если пароли не совпадают, тогда показывается notification “Пароли не совпадают”. Ввести менее 4-х символов невозможно, срабатывает проверка, и показывается notification “Пароль должен быть минимум 4 символа” По нажатию на “Сохранить” перезаписываем пароль для входа При следующем запуске приложения, не показывается экран авторизации - данные сохраняются. Только если человек разлогинился, и снова авторизовался с временным паролем,показывается экран смены пароля.
